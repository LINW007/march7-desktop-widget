# /novel:write <章号> — 章节创作（12 步融合流水线）

执行完整的写章流水线：从上下文检索到去 AI 润色到质量门控，一条龙完成。

## 适用场景
- 写新章节
- 重写不达标的章节
- 从断点继续写作

## 前置条件
- 目标章节的 L3 契约已存在（`/novel:plan` 已生成）
- 如是续写，前文章节已完成

## 12 步流水线

```
┌─────────────────────────────────────────────┐
│ Step 1  预检                                 │
│         项目根有效 → Story System 健康        │
│         → 占位符 OK → 依赖就绪                │
├─────────────────────────────────────────────┤
│ Step 2  合同刷新                             │
│         加载/刷新 runtime contract           │
├─────────────────────────────────────────────┤
│ Step 3  上下文组装 ← ContextAgent            │
│         RAG检索 + 伏笔 + 角色状态 + 风格参数  │
│         → 输出：创作任务书                    │
├─────────────────────────────────────────────┤
│ Step 4  起草正文 ← ChapterWriter Phase 1     │
│         基于任务书起草 ≥2000 字               │
├─────────────────────────────────────────────┤
│ Step 5  多维审查 ← Reviewer                  │
│         统一十维评分 + blocking issue 检测    │
│         blocking → 退回 Step 4               │
├─────────────────────────────────────────────┤
│ Step 6  去 AI 润色 ← ChapterWriter Phase 2   │
│         六层结构规则 (L1-L6)                  │
│         + 人性化技法 (≥3种)                   │
│         + C23 内心活动锚点                    │
│         + C24 结构呼吸感                      │
├─────────────────────────────────────────────┤
│ Step 7  质量门控 ← Reviewer 终审             │
│         ≥4.0 通过 / ≥3.5 二次润色            │
│         / ≥3.0 修订 / ≥2.0 裁决 / <2.0 重写  │
├─────────────────────────────────────────────┤
│ Step 8  事实提取 ← DataAgent                 │
│         实体→index.db / 伏笔→state.json       │
├─────────────────────────────────────────────┤
│ Step 9  摘要生成 ← Summarizer                │
│         章节摘要 + 状态增量 + 串线检测        │
├─────────────────────────────────────────────┤
│ Step 10 提交生成                             │
│         CHAPTER_COMMIT + audit 写入           │
├─────────────────────────────────────────────┤
│ Step 11 投影更新                             │
│         state/index/summary/memory/vector    │
├─────────────────────────────────────────────┤
│ Step 12 备份 + 输出报告                      │
│         章节备份 + 写后报告                   │
└─────────────────────────────────────────────┘
```

## 使用示例
```
/novel:write 1       # 写第1章
/novel:write 45      # 写第45章
```

## 断点续写
重复执行同一章号时，系统自动从上次断点继续：
- 如果 Step 4 已完成但 Step 5 未通过 → 从 Step 4 修正开始
- 如果 Step 6 已完成 → 直接跳到 Step 7
- 如果全部完成 → 提示"本章已写完，使用 /novel:write {下一章} 继续"

## 重写
如果需要完全重写某章：
```
/novel:write 5 --rewrite
```
这会清空该章的所有中间产物，从头开始。
