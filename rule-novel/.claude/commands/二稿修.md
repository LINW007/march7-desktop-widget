# /二稿:修 <范围>

二稿深度修订。**Claude是作者——trim-master的建议是参考，我的判断是最终。**

## 每批流程

### 第0步：重读自己的稿子
- 我通读本批全部章节——不靠Agent，靠自己的眼睛
- 标记"我自己觉得哪里不对"——这个列表先于任何Agent报告
- Agent没发现但我发现了 → 修。Agent发现了但我觉得没必要 → 不修

### 第1步：全Agent审查（6个并行）
- beta-reader + pace-editor + de-ai-checker
- continuity-auditor + chapter-validator + cost-ledger(完整)
- Agent报告是第二意见——不是判决书

### 第2步：我的裁决
- 读 trim-master 的执行清单（P0/P1/P2/P3）
- **我逐条判断**：同意/不同意/换一种修法
- 我和 trim-master 意见不同 → 以我为准
- 合并我自己的阅读标记 + Agent 发现 → 出最终修改清单

### 第3步：动手修
- P0（阻塞）→ 我亲自动手。不委托 Agent
- P1（重要）→ 我改，改完自读一遍
- P2+P3 → 判断哪些值得修。可以不修

### 第4步：修后自检
- 修改后的章节通读一遍
- 确认没引入新问题
- 确认章纲没偏
