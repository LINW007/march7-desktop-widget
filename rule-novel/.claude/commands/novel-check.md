# /novel:check — 小说 Bug 自检

对指定章节或全文执行系统性 Bug 检查，基于 novel-bug-checker 的分析框架。

## 用法

```
/novel:check <章节文件路径>    # 单章检查
/novel:check --all              # 全文检查
/novel:check --ch 1-4           # 范围检查
```

## 检查维度

### 1. 逻辑漏洞
- 时间线矛盾：事件发生顺序是否合理
- 因果断裂：关键转折是否有前置条件
- 能力突变：角色能力是否在未解释的情况下变化
- 信息知晓：角色是否知道不应该知道的信息
- 设定冲突：两个规则/设定是否互相矛盾

### 2. 角色一致性
- 性格突变：角色行为是否偏离已建立的人设
- 动机矛盾：目标与行动是否一致
- 对话风格：语言是否符合角色背景
- 肢体标记：角色的标志性动作是否前后统一（如陈渡的右手食指伤口、老周的搪瓷杯）

### 3. 叙事节奏
- 信息密度：是否有信息过载或过于稀疏的段落
- 场景过渡：切换是否自然
- 高潮铺垫：紧张感积累是否充分

### 4. 伏笔追踪
- 已埋设的伏笔是否回收
- 已回收的伏笔是否有设置
- 是否有"空头支票"

### 5. 细节一致性
- 物品位置（规则纸在哪、手机电量等）
- 身体状态（伤口状态、疲劳程度）
- 环境细节（时间、光线、天气）

## 输出格式

```
## 🔍 Bug 检查报告 — <章节名>

### 问题统计
- 🔴 致命：X个
- 🟠 严重：Y个
- 🟡 中等：Z个
- 🟢 轻微：W个

### 问题详情

#### Bug #1
- **位置**：第X行
- **严重程度**：🔴 致命
- **问题描述**：...
- **证据**：...
- **修复方案**：
  - 方案A（推荐）：...
  - 方案B：...
```

## 与 novel-fusion Reviewer 的分工

| | /novel:review | /novel:check |
|---|---|---|
| 关注点 | 这一章写得好不好 | 这一章有没有逻辑漏洞 |
| 方式 | 十维评分 + 门控 | 逐条对照 + 分级 |
| 输出 | 通过/重修/重写 | Bug 清单 + 修复方案 |

两者互补，建议每章写完后先 `/novel:check` 修 Bug，再 `/novel:review` 评分。
