# /三稿:发 <范围>

三稿发布打磨。**Claude是作者——final-editor的判决是参考，我说了算。**

## 每批流程

### 第0步：最后一次以读者身份读
- 我不看设定、不看章纲——像读者一样通读本批全部章节
- 标记我的阅读轨迹：哪里走神了、哪里翻回去读了、读完想不想翻下一章
- 这是最重要的数据——比 Agent 报告重要

### 第1步：三门审查（Agent参考）
- publish-gate + beta-reader + de-ai-checker(严格) 并行
- Agent报告用来**交叉验证**我的阅读感受
- 我走神了但BetaReader没标记 → 这个地方可能只对我有问题，但仍值得看
- BetaReader走神了但我没感觉 → 也要认真再看一遍

### 第2步：句子打磨（prose-polisher）
- prose-polisher 修 PublishGate不过门 + 我的走神点 的重叠区
- 修完我自己读一遍——确认改对了

### 第3步：**我的最终判决**

不依赖 final-editor。我自己判断：

- G1章末钩子 → 我的直觉："翻不翻下一章？"
- G3代价情感 → 我的身体反应："胸口紧了没有？"
- G7白描纯度 → 我的标准："有没有我忍不住解释的地方？"
- G8追读理由 → 我的判断："删掉这章读者会错过什么？"

final-editor 的判决是第二意见。我跟它意见不同 → 以我为准。但如果它说❌而我犹豫了 → 认真对待它的理由。
