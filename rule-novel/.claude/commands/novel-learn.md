# /novel:learn <经验> — 项目学习

将写作过程中的经验、教训、有效技法存入项目长期记忆。

## 适用场景
- 发现某个钩子设计特别有效 → 记录以便复用
- 某个套路读者反馈好 → 标记为"已验证有效"
- 某种写法翻车了 → 标记为"避免使用"
- 找到了适合本书的节奏公式 → 沉淀为项目知识

## 使用示例
```
/novel:learn "第3章的'意外重逢'钩子效果很好，读者评论说停不下来。模式：期待建立→意外阻断→信息不对称→承诺下一章揭示。"

/novel:learn "本书读者不喜欢长时间的内心独白，超过200字的独白会导致追读率下降。"

/novel:learn "修仙等级之间的战力差建议用'段位差=人数碾压'来表现：高一段位=同时打10个低段位。"
```

## 记忆结构
每条学习记录包含：
```json
{
  "id": "LEARN-XXX",
  "timestamp": "ISO时间戳",
  "chapter": "来源章节",
  "category": "技法/套路/教训/读者偏好/节奏",
  "content": "经验内容",
  "evidence": "支撑证据（评论数据/追读数据/自我判断）",
  "status": "已验证/待验证/已失效"
}
```

## 存储位置
写入 `.webnovel/memory_scratchpad.json` 的 `learned_lessons` 字段。

ContextAgent 在生成任务书时自动检索相关学习记录并注入。
