# 写后检查清单

> **用途**：每写完一批章节后逐项勾对。不依赖 Agent——Claude 自己跑。
> **原则**：5 项全部 ✅ 才能说"这批写完了"。

---

## 检查项

### 1. 钩子追踪

- [ ] 本章新埋的钩子已录入 `钩子追踪.md`（ID: HK-xxx）
- [ ] 本章兑现的钩子已从"待兑现"移到"已兑现"
- [ ] 钩子统计数字已更新（埋设数/兑现数/待兑现数）
- [ ] 章末钩子链已更新（本章结尾钩子 → 下一章预期回应）

### 2. 代价账本

- [ ] 白线物理进展：如有变化→记录位置+触发事件；如无变化→标注"本章无进展"
- [ ] 四滴传承：如有新发现→更新对应 drop 的 current_state
- [ ] 伤口七模式：如有新模式解锁或模式切换→记录
- [ ] 硬抄本规则：如有褪色进展或新规则写入→更新 status
- [ ] 代价哲学：如有新理解→记录 milestone

### 3. 连续性快照

- [ ] 时间线：故事天数/时间跳跃已更新
- [ ] 角色状态：每个出场角色的 location/physical_state/emotional_state/possessions 已更新
- [ ] 伏笔热度：foreshadow_heat 中已兑现的条目已移除，新埋的已添加
- [ ] last_chapter 和 last_updated 已更新

### 4. 全局状态

- [ ] `state.json` 的 last_chapter 已更新
- [ ] 如有新的世界观揭示→已追加到 `worldview_updates`
- [ ] 角色状态摘要已更新
- [ ] 读者期待表（reader_expectations）已刷新：新建立的期待→添加；已满足的→移除

### 5. 世界观同步

- [ ] 如有重大新揭示（新概念/新规则/新层面）→已同步到桌面 `世界观/世界观.md`
- [ ] 如有角色重大状态变化→已同步到对应角色文件（`陈渡.md` 等）
- [ ] 如有流程/方向决策→已记录到 `AI决策记录/决策记录.md`

## 常见遗漏提醒

- **白线位置**是最容易漏的——每章至少看一眼陈渡的右手食指
- **钩子ID编号**容易跳号——新增钩子前先确认上一个 ID 是多少
- **妈妈的脸**——ch15 后她每分钟换一张脸，如果正文提到要记录是哪张
- **陈屿的不可感知状态**——ch14 后他回到正面，和普通人交互时是否需要处理
- **硬抄本第 11/12 条**——书写者和妈妈在 ch15 写入的，后续章如果需要新规则从 13 开始
