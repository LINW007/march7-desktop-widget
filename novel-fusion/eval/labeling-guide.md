# 质量评估标注指南

## 目的

为 Reviewer Agent 的十维评分体系提供人工标注标准，用于：
- 校准 QualityJudge 的评分精度（Pearson 相关系数）
- 构建回归测试数据集
- 持续改进评分标准

## 标注流程

### 1. 选取样本章节
- 从已完成的章节中按质量分层抽样
- 确保覆盖：高分章（≥4.0）/ 中等章（3.0-3.9）/ 低分章（<3.0）
- 每卷至少抽 3 章（开头 / 中间 / 结尾）

### 2. 独立评分
- 至少 2 位标注者独立评分（不互相参考）
- 使用与 Reviewer 完全相同的十维评分标准
- 每个分数必须附带正文引用作为证据

### 3. 计算一致性
- 标注者间一致性（Inter-Annotator Agreement）：目标 ≥ 0.7 (Krippendorff's alpha)
- 标注者与 AI 评分一致性（Pearson r）：目标 ≥ 0.7

### 4. 不一致处理
- 两位标注者分差 > 1.0 的维度 → 讨论后重新评分
- AI 评分与人工均值分差 > 1.0 → 检查评分标准是否需要调整

## 校准脚本使用

```bash
# 运行校准（需至少 30 个标注样本）
bash eval/scripts/calibrate-judge.sh

# 输出：
# - Pearson r（AI vs 人工）
# - 各维度偏差分析
# - 阈值建议（当前门控值是否需要调整）

# 运行回归测试
bash eval/scripts/run-regression.sh

# 输出：
# - 合规率（通过门控的章节占比）
# - 评分汇总
# - 与上次运行的对比
```
