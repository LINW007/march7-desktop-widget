# /novel:review <范围> — 质量审查

对已完成或正在写的章节进行统一十维审查。

## 适用场景
- 写完后质量把关
- 批量审查已完成的章节
- 检查修改后的章节是否达标
- 定期质量审计

## 前置条件
- 目标章节正文存在
- L3 契约文件存在
- 风格指纹存在

## 执行流程

调用 **Reviewer Agent（统一审查官）**：

### 审查内容（十维评分）
1. 情节逻辑 (14%)
2. 角色塑造/OOC (14%)
3. 一致性 (12%)
4. 风格自然度 (12%)
5. 沉浸感 (10%)
6. 节奏控制 (10%)
7. 爽点密度 (8%)
8. 追读力 (8%)
9. 伏笔处理 (6%)
10. 情感冲击 (6%)

### Blocking Issues
- L1 硬规则违反
- 严重 OOC
- 情节逻辑断裂
- 时间线/设定矛盾
- 契约 forbidden 触犯

### 门控决策
| 分数 | 决策 |
|------|------|
| ≥ 4.0 | ✅ 通过 |
| ≥ 3.5 | 🔄 二次润色 |
| ≥ 3.0 | 🔧 自动修订 |
| ≥ 2.0 | 👤 作者裁决 |
| < 2.0 | ❌ 强制重写 |

### 双裁判
- 卷首章/高潮章/收尾章 → Sonnet + Opus 独立评分取均值
- 分歧 > 0.5 → 标注并报告

## 使用示例
```
/novel:review 5           # 审查第5章
/novel:review 1-10        # 审查第1-10章
/novel:review 3,7,15      # 审查第3、7、15章
/novel:review last        # 审查最新一章
```

## 产出
- 结构化审查报告 JSON（存入 `审查报告/`）
- 如为批量审查 → 额外生成质量趋势图数据
