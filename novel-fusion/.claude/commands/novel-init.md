# /novel:init — 深度初始化

启动一个新小说项目的完整初始化流程。

## 适用场景
- 新开一本书
- 已有想法但需要系统化整理
- 参考了某部作品想学习其技法

## 执行流程

### Phase 1: 信息收集
调用 **WorldBuilder Agent**，执行 7 步交互收集：

```
Step 0: 预检工作区
Step 0.5: 灵感来源询问（可选参考作品拆解）
Step 1: 故事核与商业定位
Step 2: 角色骨架与关系冲突
Step 3: 金手指与兑现机制
Step 4: 世界观与力量规则
Step 5: 创意约束包生成（2-3 套方案供选择）
Step 6: 一致性复述与确认
🚪 充分性闸门（全部确认后进入 Phase 2）
```

### Phase 2: 生成
通过闸门后自动执行：
1. 创建目录结构（`大纲/` `设定集/` `正文/` `审查报告/` `风格/`）
2. 生成所有设定文件
3. 生成 L1 硬规则 + L2 角色契约
4. 生成风格指纹模板 + AI 黑名单
5. 生成 Story System 合同种子
6. 生成总纲（`.story-system/seed.json`）

### Phase 3: 报告
输出三段式初始化报告。

## 使用示例
```
/novel:init
```
然后按提示逐步填写信息。

## 注意事项
- 这是**深度初始化**，预计需要 15-30 分钟
- 信息越详细，后续写作越顺畅
- 可以随时说"跳过"来跳过非必填项
